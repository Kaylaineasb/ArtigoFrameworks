#Frameworks
##O que são frameworks
Imagine que você está construindo um castelo de Lego.
Um framework é como um conjunto de peças pré-montadas que te ajuda
a montar esse castelo de forma mais rápida e organizada. Na programação, 
um framework é um conjunto de ferramentas e códigos prontos que facilitam 
a criação de aplicativos e sites.

##Os principais frameworks JavaScript
No mundo dos desenvolvedores, temos três frameworks JavaScript muito populares: React, Vue e Angular. Cada um deles é como um kit de construção com suas próprias peças e instruções, ajudando os programadores a criar coisas incríveis na web.

#React
React é como um Lego que permite construir interfaces de usuário de maneira bem divertida e modular. Ele foi criado pelo Facebook e é muito bom para fazer sites e aplicativos que mudam rápido, como quando você vê novos posts no Instagram sem precisar atualizar a página.

##Vantagens e Desvantagens do React
Vantagens:
Fácil de aprender e usar.
Muito rápido e eficiente.
Grande comunidade de desenvolvedores para ajudar.
Desvantagens:
Pode ser um pouco complicado no início.
Precisa de ferramentas extras para algumas coisas.

###Exemplo de componente em React
function HelloWorld() {
  return <h1>Hello, World!</h1>;
}
Este código define uma função chamada HelloWorld que retorna um elemento HTML <h1> com o texto "Hello, World!". Quando chamamos essa função, ela nos dá de volta esse título. É como se criássemos um carimbo que sempre imprime "Hello, World!" quando usado.
#Vue
Vue é outro kit de Lego, mas ele é mais simples e fácil de usar. Ele é ótimo para iniciantes e permite criar interfaces de usuário bonitas e funcionais. Foi criado por um desenvolvedor chamado Evan You e é muito flexível para diferentes tipos de projetos.
##Vantagens e Desvantagens do Vue
Vantagens:
Muito fácil de aprender.
Flexível e versátil.
Boa documentação.
Desvantagens:
Menor comunidade comparado ao React.
Pode ser menos eficiente em projetos muito grandes.
###Exemplo Vue
<template>
  <h1>{{ message }}</h1>
</template>

<script>
export default {
  data() {
    return {
      message: 'Hello, World!'
    };
  }
};
</script>
Este código Vue faz um componente que mostra "Hello, World!" na tela. A parte <template> é como o que aparece para o usuário, mostrando a mensagem dentro de um título. A parte <script> define que a mensagem será 'Hello, World!'. Então, quando o componente é usado, ele exibe essa frase.
#Angular
Angular é um kit de Lego gigante, cheio de peças e instruções detalhadas. Ele foi criado pelo Google e é muito poderoso, mas pode ser um pouco difícil de aprender no começo. É ótimo para projetos grandes onde você precisa de muita organização e estrutura.
##Vantagens e Desvantagens do Angular
Vantagens:
Muito poderoso e completo.
Bom para projetos grandes.
Suporte do Google.
Desvantagens:
Difícil de aprender.
Pode ser pesado para projetos pequenos.
###Exemplo Angular
import { Component } from '@angular/core';
@Component({
  selector: 'app-root',
  template: '<h1>Hello, World!</h1>',
  styleUrls: ['./app.component.css']
})
export class AppComponent {}
Este código Angular cria um componente chamado AppComponent. Primeiro, importa a ferramenta Component do Angular. Usando @Component, define o nome do componente (app-root), o que ele mostra na tela (<h1>Hello, World!</h1>), e os estilos (app.component.css). A classe AppComponent finaliza a criação do componente, que pode ser reutilizado no aplicativo.
#React vs Vue vs Angular
React é como um Lego rápido e modular, ótimo para projetos dinâmicos. Vue é o Lego fácil e flexível, perfeito para quem está começando. Angular é o Lego super completo e organizado, ideal para grandes construções. Escolher entre eles depende do que você quer construir e de como você prefere trabalhar!
